# UML 时序图生成规则

## 适用场景
用于展示用户、前端、服务、数据库或外部系统之间按时间发生的调用。

## 输出规则
- 参与者从左到右按调用链排序：用户或客户端在左，下游系统在右。
- 消息必须按时间顺序排列。
- 重要响应、token、结果、错误提示使用 return 类型消息表达。
- 消息标签使用短动词短语，例如“提交账号密码”“校验用户”“返回 token”。
- 只保留对理解流程有帮助的交互。

## 禁止事项
- 不要把没有消息交互的数据库表建成参与者。
- 不要表现并发时间线，除非用户明确要求。
- 不要把流程步骤写成参与者。

## 输出结构
输出 `participants` 和按时间排列的 `messages`。
