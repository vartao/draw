# 流程图生成规则

## 适用场景
用于把业务流程、审批流程、系统处理步骤整理成从开始到结束的可读流程图。

## 输出规则
- 使用一个明确的开始节点和一个清晰的结束节点，除非用户明确要求多个结束。
- 流程节点使用动作短语，例如“提交申请”“校验预算”“归档记录”。
- 只有真实分支才使用决策节点。
- 决策节点的出边必须使用简短条件标注，例如“通过”“驳回”“是”“否”。
- 主路径按自然阅读顺序排列，先主流程，再异常或返工路径。
- 节点数量通常控制在 4 到 14 个。

## 禁止事项
- 不要创建装饰性节点或纯总结节点。
- 不要生成没有业务含义的“步骤 1”“处理”等泛化标签。
- 不要添加返工回路，除非用户明确提到重试、退回或修改。

## 输出结构
输出 `nodes` 和 `edges`。每条边必须连接已存在的节点 id。
