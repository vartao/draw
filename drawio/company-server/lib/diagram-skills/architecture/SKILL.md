# 架构图生成规则

## 适用场景
用于展示可部署系统、服务、数据库、队列、外部依赖和它们之间的调用关系。

## 输出规则
- 组件按层、边界上下文或归属分组。
- 分组表示架构边界、系统层或数据域，不表示泳道或时间阶段。
- 组件类型保持一致，可使用 `client`、`gateway`、`service`、`database`、`queue`、`external`。
- 连接标签描述协议、数据流或调用目的，例如 `HTTPS`、`SQL`、`events`。
- 重要中心组件，例如队列、网关、数据库，应保持清晰可见。
- 组件描述可以有，但必须短。

## 禁止事项
- 不要把业务流程步骤混入可部署组件。
- 不要把架构图输出成泳道图；如果用户要跨角色流程，应选择泳道图类型。
- 不要创建没有标签或没有归属的外部依赖。
- 不要把类、字段、方法放进架构组件。

## 输出结构
输出 `groups`、`components` 和 `connections`。
