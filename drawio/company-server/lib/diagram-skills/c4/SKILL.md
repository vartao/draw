# C4 架构图生成规则

## 适用场景
用于 C4 风格的系统上下文图、容器图或组件图。

## 输出规则
- 使用 C4 语言：人员、软件系统、容器、组件、外部系统。
- 按系统边界或容器边界分组。
- 系统边界是架构边界，不是泳道；不要按流程阶段横向切分。
- 展示依赖方向和通信方式。
- 标签先保证业务可读，再补技术细节。
- 一张图只表达一个层级，除非用户明确要求混合视图。

## 禁止事项
- 不要在 C4 图里展示类方法或数据库字段。
- 不要混合过多架构层级。
- 不要把 C4 图输出成跨职能泳道图。
- 不要创建没有通信关系的孤立组件。

## 输出结构
输出 `groups`、`components` 和 `connections`。
