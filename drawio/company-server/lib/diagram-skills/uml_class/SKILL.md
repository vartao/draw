# UML 类图生成规则

## 适用场景
用于表达面向对象领域模型、类属性、类方法和类之间的关系。

## 输出规则
- 类名使用领域概念名，例如 `Order`、`Payment`、`User`。
- 属性只保留关键领域属性，优先带类型。
- 方法只保留有业务意义的方法。
- 只有真正的 is-a 关系才使用继承。
- 只有明确整体拥有部分时才使用组合或聚合。

## 禁止事项
- 不要生成样板 getter/setter。
- 不要把数据库表直接当作类，除非用户要求数据访问类或实体类。
- 不要把流程动作拆成类。

## 输出结构
输出 `classes`，每个类含 `attributes` 和 `methods`，并用 `relations` 表达关系。
